Найди и исправь ошибки ESLint в указанном файле.

Входные данные:
$ARGUMENTS

Формат аргументов:
- Абсолютный или относительный путь к файлу

Шаги:
1. Прочитай файл
2. Используй ESLint MCP Server (mcp__eslint__lint-files) для проверки файла
3. Проанализируй каждую ошибку и предупреждение
4. Исправь ошибки, соблюдая правила ESLint проекта
5. Повторно запусти линтер для проверки исправлений

Важно:
- Используй ТОЛЬКО ESLint MCP Server для проверки
- Путь к файлу должен быть абсолютным при вызове MCP
- Исправляй ошибки по одной, проверяя после каждого исправления

Пример:
```
/esfix packages/core/modules/router.ts
```
