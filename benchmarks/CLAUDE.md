It's a new metric в†’ existing results lack it until the next full run repopulates. ~60 navs/pass в†’ ~28-1000 samples depending on engine; the warmup+K median stabilizes it (RME < 15%).

Capability matrix (`вњ“вЃ±` = verified in-harness via a functional demo app under `apps/react/<engine>/{data,search,guard}/`) shows feature gaps a perf table can't вЂ” among the full routers, `react-router` lacks first-class validated search. `wouter` is excluded from the roster (minimalist location-matcher, different class + no cross-framework analog). Per-cohort, never cross-framework.

**Per-cohort only вЂ” no cross-framework ranking** (a cross-framework number is mostly a framework comparison). CPU (`script`) + heap are the stable signals; latency is paint-noisy; `nav-churn.navsPerSec` is frame-capped (read CPU/nav + heap). Driven via the `playwright` package (programmatic Chromium + `CDPSession`), not `@playwright/test`. Results: `cross-router/results/` (feeds the infographic deck); design: [`.claude/cross-router-benchmarks-design.md`](.claude/cross-router-benchmarks-design.md).

**Sweeps measure WARM steady state, not the cold first nav (#1453).** `measure.mjs` runs each sample in a fresh context, and every sweep `land()` is a full `page.goto` reload that resets V8 to interpreted code вЂ” so a single post-reload nav timed the *interpreter*, and the per-engine cold penalty (larger for heavier bundles) poisoned cross-engine `@N` absolutes (measured: solid real-router carried ~0.3 ms of extra interpreter cost over @solidjs/router that vanished when warmed). `wide-config`/`deep-config`/`search-param-scaling` now warm the realm with `WARM_NAVS=12` in-document navs before the ONE measured nav вЂ” wide/search toggle `targetв†”pivot` via the persistent nav; deep round-trips `homeв†”D`, detecting the return home by the target *vanishing* (`__navMetric.settleGone`, the mirror of `settle`, since deep's home has no positive universal marker). The matcher/parse/composition **curve** is source-independent and unchanged; only the cold-JIT floor is removed, so `@N` absolutes dropped (floor-dominated sizes ~85%: wide@1000 rr 0.66в†’0.10 ms script; composition-dominated deep@90 only ~11% вЂ” its cost is real render work). Validate with `harness/f3-warm-validate.mjs <engine> [fw] [variant] [target]` (same-session cold-vs-warm, no results/ write).

**Sweep HEADLINE is `navMsTask@N` + `navMsWall@N`, not `scriptMs@N` (#1451, closing F2 in the sweeps too).** `ScriptDuration` is V8-only and blind to async engines' microtask composition, so the old `scriptMs@N` / `totalMs@N = script+blink` printed a FALSE flat curve for @solidjs/router / vue-router / @tanstack (measured: solid-router deep@90 `scriptMs` **0.035** вЂ” near-flat artifact вЂ” vs `navMsTask` **0.838**, a **24Г— undercount**; on the honest task metric solid-router is even *heavier* than rr at depth 90, the opposite of the scriptMs story the audit flagged as instrumental). The measured nav is now timed clickв†’settle (`perf.now` = `navMsWall`, `О”TaskDuration` = `navMsTask`, both microtask-inclusive); `scriptMs@N` stays a вљ  diagnostic, `totalMs@N` is retired. `navMsTask@N` **leads** over `navMsWall@N` because the single-nav `perf.now` is ~100 Вµs clamp-quantized (the per-nav scenarios sum N navs so their wall is clean; sweeps measure one nav per size). So `navMsWall@N` is emitted at the **endpoint only** (matching the report row) and `rme-gate` classes `navMsWall@N` **noisy** вЂ” a lean engine at the endpoint (rr wide@1000 в‰€ 0.1 ms = 1 clamp step) has a high RME by *quantization*, not instability, and must not fail a healthy 4-h matrix; `navMsTask@N` (unclamped) stays stable-gated as the headline. **Consequence: the sweep metric KEYS changed в†’ REPORT sweep tables render `вЂ”` and the blurbs' absolutes are stale until the next re-baseline** (unlike the F3-only change, which kept keys).

**Smoke-grade cells never reach `results/` (#1455).** `run.mjs` refuses to write a cell when `runs < N_MIN` (=10) вЂ” it still builds/measures/prints (so `--smoke` fail-fast AND same-session A/B at nв‰Ґ12 both work), but a low-n cell's reported RME (`stats.mjs` z=1.96) badly understates its true small-sample CI, and cross-session sub-ms drift can *invert* a О” вЂ” so it must not sit in `results/` indistinguishable from a real cell. `rme-gate` also FAILS on any pre-existing `n < N_MIN` cell (defense-in-depth; env `RME_MIN_N`). **Corollary вЂ” `О”(fullв€’bare)` is same-session only:** the `real-router-full` variant prices the production plugin stack (persistent-params + search-schema + ssr-data over the base app); measure its bare/full pair **back-to-back in one session** (`run.mjs nav-latency real-router react 50 && run.mjs nav-latency real-router-full react 50`) вЂ” a cross-session pair produced a physically-impossible **в€’51%** О” (full cheaper than a strict superset) purely from ~2Г— sub-ms drift вЂ” the "same-session A/B only for sub-ms" rule, caught live (#1455).

**The matrix INTERLEAVES engines and refuses a stale dist (#1459 / #1460).** `run-all.mjs` no longer measures one engine's cell then the next вЂ” for each scenario it builds + serves ALL of a cohort's engines and measures them ROUND-ROBIN in one browser session (`measureInterleaved`, rotating the order each round), so mid-run machine drift hits every engine equally (no engine is systematically first вЂ” the S4 position-bias class a fixed roster left open). Before any build both runners call a shared dist-freshness gate (`harness/provenance.mjs`): if `max(mtime packages/*/src) > max(mtime packages/*/dist)` it **REFUSES** (`exit 3`, override `BENCH_ALLOW_STALE_DIST=1`) вЂ” engines resolve to `dist/`, and the 07-11 vue matrix silently measured a pre-#1424 dist. Every cell's `env` now carries `{commit, dirty, distNewestMtime}`. `run.mjs` (one cell) + `run-all.mjs` (interleaved matrix) share the gate, the provenance stamp, the smoke-grade write guard, and the scenario/variant registry (`harness/{provenance,write-cell,scenarios-registry}.mjs`) so the two can't drift.

**Engines are measured as PRODUCTION `dist`, not `src` вЂ” fairness-critical, verified.** `run.mjs` does `vite build` + `preview` (a STATIC production bundle, **no on-the-fly compilation** вЂ” unlike dev-mode or the core `mitata` benches where `tsx` transpiles TS live and that cost pollutes results). vite resolves `@real-router/*` via the default `import` condition в†’ `packages/*/dist/esm/index.mjs`; the `@real-router/internal-source`в†’`src/*.ts` condition is a *scoped* condition **invisible to vite** (only `tsc`/vitest activate it вЂ” **0** bench vite-configs add it). Competitors resolve to their published `node_modules` dist. Both real-router dist + competitor dist are esbuild-minified into the app bundle (`minify: esbuild`) в†’ **fair dist-vs-dist, real shipped bytes**. **Empirically confirmed** (2026-07-02) via `vite pluginContainer.resolveId`: `core`/`react`/`browser-plugin` all в†’ `packages/*/dist/esm/index.mjs`. Consequence: `cold-start`'s core-parse slice is real-router's *real* production-bundle load (a core-size / eager-init characteristic вЂ” a legit optimization axis, e.g. trim/lazy-init core), **NOT** a src-compilation artifact. **After any `packages/*/src` edit, re-`pnpm bundle`** or the bench measures the stale dist.
