# Invariants

> Property-based invariants verified via [fast-check](https://fast-check.dev/). See `tests/property/` for implementations.

## Parse / Build

| #   | Invariant                                           | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| --- | --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Roundtrip (string values)                           | `parseQuery(build(params, opts), opts) === params` for string-only values. Verifies the core serialization cycle is lossless.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| 2   | Roundtrip with type normalization                   | `parseQuery(build(params, opts), opts) ≈ normalizeForComparison(params, opts)` for mixed-type values. Numbers and booleans serialize as strings, so comparison uses type-normalized expected values.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| 3   | Double-roundtrip stability                          | `parseQuery(build(parseQuery(build(p, opts), opts), opts) , opts) ≡ parseQuery(build(p, opts), opts)`. A second pass through the build/parseQuery cycle produces no further changes.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 4   | Build determinism                                   | Two calls to `build(params, opts)` with identical arguments always return the same string. The function has no hidden state.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| 5   | Boundary value acceptance                           | `parseQuery("")` returns `{}`, `parseQuery("?")` returns `{}`, and `build({})` returns `""`. Empty inputs are handled without errors.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| 6   | No `?` prefix from build                            | `build(params, opts)` never returns a string starting with `?`. Callers are responsible for prepending the separator when needed.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| 7   | Percent-encoding roundtrip                          | **Keys and values** containing special characters (spaces, `&`, `=`, `?`, `#`, `+`, `/`) and multibyte/unicode content survive the `build → parseQuery` cycle losslessly via `encodeURIComponent`/`decodeURIComponent`. Includes keys that shadow `Object.prototype` members (`valueOf`, `constructor`, `toString`, `__proto__`), which decode to plain own properties, not the inherited functions.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| 8   | Undefined exclusion                                 | `build(params)` silently drops keys whose value is `undefined`, producing the same output as building without those keys.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 9   | Empty array erasure                                 | `build({key: []})` produces an empty string for that key (all formats including comma), so `parseQuery(build({key: []}))` does not contain `key`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| 10  | No-options defaults symmetry                        | `parseQuery(build(params)) ≈ normalizeForComparison(params, {})` — without options, `build` and `parseQuery` resolve to the same cached `auto` defaults, so the roundtrip holds with no options passed. (#744)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 11  | String boolean-token coercion                       | A string value equal to `"true"`/`"false"` decodes to a **boolean** under `booleanFormat: "auto"`/`"empty-true"` — the word-match reads the **raw** (pre-percent-decode) value (`decodeRaw`, before the number/string strategies). So an over-encoded spelling (`%74rue`) stays the **string** `"true"`, whereas `number` coerces on the **decoded** value (`4%32` → `42`): a deliberate raw-vs-decoded asymmetry, locked in `inverse-pair-brutal.properties.ts` (#1317). Guards the property-oracle from silently mis-modelling this. (#746/#1317)                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 12  | Inverse-pair totality (`range(parseQuery) ⊆ dom(build)`) | `build(parseQuery(qs, opts), opts)` **never throws** for any wire string `qs` **in `dom(parseQuery)`** over the full option matrix. A **lone (unpaired) surrogate** — accepted by `parseQuery` (non-percent decode is identity) but rejected by `encodeURIComponent` — is kept total by `safeEncode`, which sanitizes it to U+FFFD (`%EF%BF%BD`) rather than throwing (#1314; total-but-lossy, Format Roundtrips #20). **Outside `dom(parseQuery)`:** malformed percent (`%zz`, `%2`, valid-hex/invalid-UTF-8 `%FF`) makes `parseQuery` **itself** throw `URIError` by design — the router guards it at `SegmentMatcher.#mergeQueryParams` (→ UNKNOWN_ROUTE, never a crash, #737), so it is not part of this contract. The wire-grammar generators (`inversePair`/`inverse-pair-brutal` `arbRawQueryString`/`arbBrutalQs`, now surrogate-bearing) reach the `parseQuery→build` half the `build`-derived generators are blind to — the class that let #1155 (null array element) crash `router.start()` on a forgeable URL. (#1155/#1314, axis A7) |
| 13  | Empty-chunk skip                                    | An empty query chunk — a `&&`, a leading `&`, or a trailing `&` — is **skipped** in `parseQuery`, not decoded into a junk `{ "": null }` param (`parseQuery("&a=1")` → `{a: 1}`, `parseQuery("x=1&&&x=2")` → `{x: [1, 2]}`). An intentional empty-key chunk carries an `=` (`parseQuery("=1")` → `{ "": 1 }`) so its span is non-empty and is unaffected. (#1156)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |

## Format Roundtrips

| #   | Invariant                                     | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| --- | --------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Array format: brackets/index                  | `parseQuery(build(params, {arrayFormat}), {arrayFormat}) === params` for `"brackets"` and `"index"` formats with string arrays.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| 2   | Array format: none (multi-element)            | Multi-element string arrays roundtrip correctly with `arrayFormat: "none"` via repeated keys.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| 3   | Array format: comma (multi-element)           | Multi-element string arrays roundtrip correctly with `arrayFormat: "comma"`. Unencoded commas are separators; encoded `%2C` stays literal. Single values without commas remain scalars.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| 4   | Boolean format: auto                          | `parseQuery(build(params, {booleanFormat: "auto"}), {booleanFormat: "auto"}) === params`. Boolean types are preserved through the cycle.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| 5   | Boolean format: empty-true (true)             | `true` values roundtrip correctly with `booleanFormat: "empty-true"` (key-only `?flag` encoding → `decodeUndefined` → `true`).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 6   | Null format: default                          | `parseQuery(build(params, {nullFormat: "default"})) === params`. `null` is encoded as a key-only entry and parsed back as `null` — **except under `booleanFormat: "empty-true"`**, which reserves the key-only form for `true` (see #18).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 7   | Null format: hidden                           | With `nullFormat: "hidden"`, `build` produces an empty string and `parseQuery` returns `{}`. Null values are fully suppressed.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| 8   | Plus-as-space equivalence                     | `parseQuery(qs.replace(/%20/g, '+'))` produces the same result as `parseQuery(qs)`. The decoder treats `+` as a space, equivalent to `%20`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| 9   | Array element type partition                  | `build({key: [v]})` succeeds for `v: string \| number \| boolean \| null` (a lone-surrogate string is sanitized to U+FFFD by `safeEncode`, not thrown — #1314/#20) and throws `TypeError` only for `v: undefined \| object`. A `null` element encodes to the bare-key form per array format (`name` / `name[]` / `name[i]` under `nullFormat: "default"`; dropped under `"hidden"` and under `comma`, which has no bare-element form), so parseQuery's null-in-array (`parseQuery("a&a=1")` → `{a: [null, 1]}`) round-trips instead of throwing (#1155).                                                                                                                                                                                                                                                                                                 |
| 10  | None format: single-element collapse          | `parseQuery(build({a: [x]}, {arrayFormat: "none"}))` returns `{a: x}` (scalar), not `{a: [x]}` (array). Single-element arrays lose their array type through the none-format roundtrip because the parser cannot distinguish a single-value key from a scalar.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| 11  | Number format: auto integers                  | `parseQuery(build(params, {numberFormat: "auto"}), {numberFormat: "auto"}) === params` for non-negative **safe** integer values. `auto` recognizes a canonical decimal number (optional leading `-`, no leading zeros except `0`, no exponent); leading-zero and unsafe-integer magnitudes are kept as strings to preserve their exact text — so `"007"` and `"9007199254740992"` do **not** become numbers. The narrowing is symmetric for negatives (invariant #14).                                                                                                                                                                                                                                                                                                                                                                              |
| 12  | Number format: auto decimals                  | `parseQuery(build(params, {numberFormat: "auto"}), {numberFormat: "auto"}) === params` for decimal values (e.g., `12.5`). Decimal numbers survive the roundtrip.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 13  | Number format: none preserves strings         | `typeof parseQuery(build({a: 42}, {numberFormat: "none"}), {numberFormat: "none"}).a === "string"`. With `numberFormat: "none"`, numbers become strings after the build/parseQuery cycle.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 14  | Number format: auto negatives                 | `parseQuery(build(params, {numberFormat: "auto"}), {numberFormat: "auto"}) === params` for negative integer values. `build({n: -5})` emits `"n=-5"` and `parseQuery` decodes it back to the number `-5`, so a param keeps the same type whether it arrives from a URL or from a programmatic `navigate({n: -5})`. Non-canonical negatives (`"-007"`, `"-9007199254740992"`) stay strings, mirroring the unsigned rules. (#742)                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 15  | Boolean format: empty-true (true & false)     | `parseQuery(build(params, {booleanFormat: "empty-true"}), {booleanFormat: "empty-true"}) === params` for **both** boolean values. `build({flag: false})` emits `"flag=false"` and `parseQuery` decodes it back to boolean `false` (not the string `"false"`); array elements (`"a=true&a=false"`) decode both `"true"`→`true` and `"false"`→`false` symmetrically. (#743)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 16  | Number format: auto non-canonical             | Under `numberFormat: "auto"`, numeric-looking strings that are **not** canonical — leading-zero (`"007"`), unsafe integers (`> 2^53`), exponent notation (`"1e5"`), and **negative zero** (`"-0"`, `"-0.0"`) — stay strings, preserving their exact text/round-trip (the inverse of #11/#14). Negative zero is a valid JS number but non-round-trippable: `String(-0) === "0"` and `build(-0)` emits `"0"`, so coercing `"-0"` would silently drop the sign. Build-side mirror: number inputs whose `String()` form is non-canonical — `NaN`, `Infinity`, `1e21` — serialize to that text (`build({a: NaN})` → `"a=NaN"`) and decode back as the **string** (`"NaN"`), a silent number→string loss on the programmatic path (correct, since `String(NaN)` is not a canonical decimal; `-0` serializes as `"0"` → decodes as `0`). (#742, #898) |
| 17  | Array format: index ordering                  | Under `arrayFormat: "index"`, `parseQuery` orders elements by the bracket index `[n]`, not arrival order: emitting `a[i]=v` chunks out of order (e.g. reversed) still parses back to the original array. Indices are sorted then compacted (a huge index like `a[1000000]` does not allocate a sparse array); non-numeric/empty brackets (`a[]`, `a[x]`) fall back to insertion order. A bare scalar chunk (`a=1`) sharing a key with indexed chunks is **displaced** by the indexed group — `parseQuery("a=1&a[0]=x", {arrayFormat: "index"})` → `{a: ["x"]}` regardless of chunk order (the indexed group replaces the scalar). (#856, #1319)                                                                                                                                                                                                          |
| 18  | Boolean/null collision: empty-true            | Under `booleanFormat: "empty-true"` the bare-key wire form `?a` is reserved for `true` (`decodeUndefined`), so a `null` value with `nullFormat: "default"` is **not representable**: `build({a: null}, {booleanFormat: "empty-true"})` emits `"a"` and `parseQuery` decodes it to `true`, not `null`. Both `null` and `true` encode to the same token and only `true` decodes back — a documented, deterministic loss (use `nullFormat: "hidden"` or avoid null under empty-true). This is asserted explicitly rather than absorbed by the roundtrip oracle.                                                                                                                                                                                                                                                                                        |
| 19  | Empty-key bare-value erasure                  | The empty-string key `""` is **erased** when its value encodes to a bare-key token (no `=`) — `true` under `booleanFormat: "empty-true"` and `null` under `nullFormat: "default"` both emit `""`, which build's empty-chunk filter drops, so `parseQuery(build(...))` loses the key (the bare-key token for `""` is `""` itself, no wire form to round-trip). Every other value (`string`/`number`/`false`) encodes `=value` and round-trips; a non-empty key's bare token (`name`) survives. The empty-key sibling of #18 (null+empty-true collapse) and Parse/Build #9 (empty-array erasure): asserted explicitly here, with the roundtrip properties excluding the two combos via `erasesEmptyKey`. Pathological — query keys come from route definitions, never empty (#1051).                                                                  |
| 20  | Lone-surrogate sanitization (total but lossy) | A lone (unpaired) UTF-16 surrogate is the only string `encodeURIComponent` rejects (`URIError`). `parseQuery` accepts one verbatim (non-percent decode is identity), so — to keep `build` total (`range(parseQuery) ⊆ dom(build)`, Parse/Build #12) — `safeEncode` sanitizes it to U+FFFD via a lone-surrogate regex (a lib-agnostic manual `toWellFormed`): `build({a: "\uD800"})` → `"a=%EF%BF%BD"`. The first round-trip mutates the (non-round-trippable) garbage — `parseQuery(build(parseQuery("a=\uD800")))` → `{a: "�"}` — then stabilises. Same class as `-0` / empty-array / empty-key losses (#16 / Parse/Build #9 / #19): asserted explicitly in `inverse-pair-brutal.properties.ts`, excluded from the exact-roundtrip oracle. path-matcher applies the identical sanitize on path params (#1315). (#1314)                                            |
